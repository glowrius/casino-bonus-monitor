User Guide

Everything you need to know about monitoring, claiming, and never missing a free SC opportunity.

Overview

Casino Bonus Monitor is a standalone Windows EXE that combines a Discord bot, Reddit RSS monitor, and auto-claimer into one background process. It scans sweepstakes casino subreddits for free SC and free spins offers, watches Kick.com streamer chats for dropped links, and can automatically claim daily bonuses from supported casinos — all delivered straight to your Discord server.

How it works
The bot polls Reddit RSS feeds every 10 seconds, filters for free SC / free spins keywords, extracts claim links from post descriptions, and posts them to your Discord channel. Streamer alerts connect via Kick's Pusher WebSocket to catch links in real-time chat. The auto-claimer runs on a cron schedule with your saved cookies.

Installation

Download the installer from the releases page and run it. The installer will guide you through Discord configuration.

System Requirements

  • Windows 10 or later (64-bit)
  • No Node.js, Python, or any runtime required
  • Internet connection (for Reddit API, Discord, Kick WebSocket)
  • ~100 MB disk space
1

Download the latest CasinoBot-Setup-v2.x.x.exe from the GitHub Releases page.

2

Run the installer. During setup you'll be asked for your Discord bot token and channel IDs. These can also be configured later in the .env file.

3

Launch CasinoBot.exe. The console will display module status and the bot will come online in your Discord server. Check with /status.

ⓘ

Running the installer with the "Install certificate" option selected will prevent Windows SmartScreen warnings on launch.

Required Intents

In your Discord Developer Portal, make sure your bot has these Privileged Gateway Intents enabled:

  • Server Members Intent — required for channel member lookups
  • Message Content Intent — required for reading message content (future use)

Commands

All commands are slash commands (type / in Discord) and registered instantly within your server.

Command Description Options
/status Show bot uptime, module status, and enabled features —
/claim Trigger an immediate daily claim run for all enabled casinos —
/setcookie Save session cookies for a specific casino <casino> name, <cookie> string
⚠

Session cookies contain your login credentials. Never share them with anyone. The bot stores them locally in data/cookies/ and only uses them for claim requests.

Cookie-Based Daily Claiming

To claim daily bonuses on your behalf, the bot needs your session cookies for each casino. Here's how to get them and use them.

Getting Your Cookies

1

Log into the casino in your browser (Chrome, Firefox, Edge).

2

Open Developer Tools (F12) → Application tab → Cookies → select the casino domain.

3

Copy the cookie string. Right-click any cookie → "Copy All" or use the document.cookie method:

javascript:prompt('Cookies:', document.cookie);

Paste this in the address bar, press Enter, and copy the full string.

4

Save to the bot with the /setcookie command:

/setcookie casino: "Chumba Casino" cookie: "sessionid=abc123; _gid=GA1.2.xyz"

Streamer Alerts

The bot can monitor Kick.com streamer chats in real-time via Pusher WebSocket. When a streamer or viewer drops a link in chat, it's forwarded to your streamer alerts channel.

How to Set Up

Edit data/streamer_profiles.json and add streamers by their Kick.com username:

[
  {
    "name": "My Streamer",
    "enabled": true,
    "kickUsername": "streamername",
    "monitorLinks": true
  }
]

Set enabled: true and the bot will connect to that streamer's chat room on startup.

How It Works

  • Bot connects to Kick.com's Pusher WebSocket for each enabled streamer's chatroom
  • Monitors ChatMessageEvent for messages containing URLs
  • Deduplicates links using SHA-256 hash (no repeats within 24h)
  • Posts alerts to your configured #streamer-chat channel
  • Auto-reconnects with a 10-second delay on disconnect
ⓘ

Streamer alerts are entirely separate from Reddit RSS alerts. They have their own channel and their own dedup database.

Auto-Claim (Scheduled)

Beyond the interactive panel, the bot can automatically claim daily bonuses on a schedule using node-cron.

Schedule

By default, auto-claim runs every day at 10:00 AM (server local time). The schedule is defined per-casino profile and can be customized.

Claim History

Each claim attempt is logged to data/claim_history.json with:

  • Casino name
  • Date and time of attempt
  • Success/failure status
  • HTTP response code
⚠

Auto-claim requires valid session cookies. Without cookies, the bot will attempt the claim but it will fail (redirected to login). Use /setcookie or the interactive panel to configure each casino.

Troubleshooting

Check the bot's console window for errors. Common causes:

  • Invalid channel IDs in .env — verify with /status
  • Reddit RSS feed changed — check REDDIT_RSS_URLS
  • Filter is too aggressive — check keyword filtering logic
  • Bot doesn't have permission to send messages in the target channel

Ensure the streamer profile is enabled ("enabled": true) and the Kick username is correct. The bot will log connection status to the console. Verify STREAMER_CHANNEL_ID is set and points to a valid channel the bot can see.

Restart the bot. If the issue persists, check the console for stack traces. Common causes: missing .env variables, corrupted data files, or network timeouts.

The bot registers commands per-guild (instant). If commands don't appear after restart, re-invite the bot with the applications.commands scope. You may need to type / and wait a few seconds for Discord to sync.

Check the console for error messages. Common causes:

  • Discord API rate limiting — bot will auto-reconnect
  • Kick WebSocket disconnects — auto-reconnects after 10s
  • Invalid data in JSON config files — validate with a JSON checker
  • Out of memory — the bot uses ~100 MB under normal load
